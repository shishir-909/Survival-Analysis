Introduction

There are times when we want to compare time-to-failure data from two or more groups. For example, a designer might want to implement a new design to reduce costs, but wants to make sure that this change does not adversely affect the reliability of the product. Or, a manufacturer might want to find out if there is any difference in the time-to-failure of products manufactured from different batches or different assembly lines. Hypothesis testing can be used to compare such data.

Two of my previous blog articles(here and here) dealt with non-parametric methods for comparing failure data. In this article, I demonstrate the use of a parametric method for the same purpose.

Data

The data used in this article is from the book Statistical Methods for Reliability Data, Second Edition (William Q. Meeker, Luis A. Escobar, Francis G. Pascual)]. Table 1 shows this data, and it is also publicly available on DataShare, Iowa State University’s open data repository which can be accessed through this link.

The values in Table 1 are the results of a pull test to measure the bond strength of wire bonds from three separate manufacturing batches. We are interested in finding out whether there are statistically significant differences in the distribution of bond strength across the different manufacturing batches. If we do find that there is a statistically significant differnce, we want to find out whre is this difference (between which pairs of batches).

Analysis with Separate Distributions

In order to proceed with a parametric comparison of the different groups, we first need to figure out which parametric distribution to use for the data. Weibull and log-normal are the most commonly used distributions to describe time-to-failure in reliability applications. For the purpose of demonstrating a parametric comparison, we will consider these two distributions. We have three groups in our dataset: Batch 1, Batch 4 and Batch 5. Generally, when the failure mode is the same in all groups, the same distribution is used to fit the data.

Figure 1 shows a Weibull probability plot and Figure 2 shows a log normal probability plot of the data. The points on these plots are the empirical CDF1 Cumulative Distribution Function and the lines are ML2 Maximum likelihood estimates from fitting the respective distributions.

xTransformer.logNormal <- scales::trans_new(
  name = "logNormal.x.transform",
  transform = function(t.p){
    x = log(t.p)
    return(x)
  },
  inverse = function(x) {
    t.p = exp(x)
    return(t.p)
  }
)


yTransfomer.logNormal <- scales::trans_new(
  name = "logNormal.y.transform",
  transform = function(p){
    y = qnorm(p)
    return(y)
  },
  inverse = function(y) {
    p = pnorm(y)
    return(p)
  }
)

Note that 3 separate Weibull distributions were fit - one for each batch. Similarly, 3 separate log normal distributions were also fit. Looking at both plots above, the Weibull distribution seems to fit slightly better than the log-normal, but not by much. The data could have come from any of the two distributions we have considered here. Like I mentioned in my previous article on parametric regression, we are mainly looking for any indication that a particular distribution is definitely not a good fit rather than proving that the data is from a particular distribution. The AIC from the Weibull model is 381.2918853 and the log-normal model is 382.2946935. Since the AIC is lower for the Weibull model, we will continue with Weibull distribution, although log-normal wouldn’t have been a terrible choice either.

Table 2 shows the location (\(\mu\)) and scale (\(\sigma\)) parameters from the log-location scale representation of the Weibull distribution.

Figure 3 shows the approximate 90% joint confidence regions based on the relative likelihood for the Weibull distribution median and the shape parameter \(\beta\). The large amount of overlap between the Batch 4 and Batch 5 regions indicates that we do not have sufficient evidence at the 10% significance level to suggest that Batches 4 and 5 have different failure time distributions. The amount of overlap for Batch 1 with the other two batches is relatively smaller, but the presence of even some overlap suggests that the difference in distributions is not statistically significant at the 10% significance level3 Notice that there is an overlap of the 90% Wald confidence intervals of the location parameter \(\mu\) in Table 2, which paints a similar picture of lack of evidence of a difference in distributions.

Although the difference is not statistically significant, we are more concerned with practical significance of the differences. Looking at the contour plot in Fig 3, it seems like batch 1 is different to batches 4 and 5. It is worth conducting pairwise comparisons between batches and the best way to do this is by constructing a confidence interval for the differences in the location parameters of the three distributions. This will be done later in the article.
